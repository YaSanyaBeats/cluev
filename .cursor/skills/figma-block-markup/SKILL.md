# Figma → HTML + SCSS (только разметка)

## Входные данные

Пользователь передаёт:

1. **Ссылку на блок** в Figma (`figma.com/design/...?node-id=...`)
2. **Путь к HTML-файлу** (например `src/html/hero.html`)
3. **Путь к SCSS-файлу** (например `src/scss/components/hero.scss`)

Если чего-то не хватает — запроси недостающее, не угадывай пути.

## Порядок работы

1. **Figma MCP** — вызови `get_design_context` (сервер `user-Figma`):
   - Из URL извлеки `fileKey` и `nodeId` (`node-id=1-2` → `1:2`)
   - `clientLanguages`: `html,scss`
   - `clientFrameworks`: `unknown` или стек проекта, если известен
   - Используй скриншот и структуру из ответа; сгенерированный React/Tailwind — только как ориентир, не копируй в проект

2. **Имя БЭМ-блока** — из имени HTML-файла без расширения (`hero.html` → блок `hero`). Корневой элемент секции: `<section class="hero">` (или семантически уместный тег + класс блока).

3. **HTML** — запиши разметку в указанный файл по правилам ниже.

4. **SCSS** — запиши пустой шаблон всех классов из HTML в указанный SCSS-файл. Стили не пиши.

5. **Проверка** — пройди [чеклист](#чеклист-перед-завершением).

## Правила HTML

Скил получает ссылку на блок из макета фигмы. А также 2 файла HTML и SCSS.

В HTML нужно записать HTML разметку исходя из figma макета. Атрибуты alt, aria-hidden и прочие атрибуты доступности писать не нужно. Атрибуты width и height также никогда использовать нельзя. Для назначения классов используй БЭМ. Каждый элемент должен иметь свой класс. Если видишь в дизайне слайдер, то используй разметку "swiper". Если любой другой интерактивный элемент, то попробуй найти его в документации UiKit

### БЭМ

- Блок: `hero`, элемент: `hero__title`, модификатор: `hero__gradient--top`
- **Каждый** DOM-элемент в разметке блока имеет свой класс (включая обёртки, строки текста, медиа)
- Не используй голые теги без класса внутри блока (кроме обязательной структуры Swiper/UIkit, см. ниже)

### Запрещено в HTML

| Запрет | Примеры |
|--------|---------|
| Доступность | `alt`, `aria-*`, `role`, `aria-hidden` |
| Размеры на тегах | `width`, `height` |

Допустимы обычные атрибуты: `href`, `src`, `class`, `type`, `for`, `name`, `placeholder`, `loading`, `decoding`, `target`, `rel` — если нужны по смыслу.

Также запрещены теги `p` и  `span`. Вместо них используй `div`.

### Слайдер (Swiper)

Если в макете карусель/слайдер — разметка Swiper 8 (как в проекте):

- На контейнер слайдера: классы блока + `swiper` (например `hero__slider swiper`)
- `swiper-wrapper` → слайды `swiper-slide` + БЭМ-класс слайда
- Пагинация/навигация: `swiper-pagination`, `swiper-button-prev` и т.д. + БЭМ-классы при необходимости

Подробный каркас: [reference.md](reference.md#swiper).

### Прочая интерактивность (UIkit)

Проект подключает `static/uikit.min.js` / `uikit.min.css` (UIkit 3).

Для аккордеона, табов, модалки, оффканваса, дропдауна, слайдера UIkit и т.п.:

1. Открой [документацию UIkit](https://getuikit.com/docs/introduction)
2. Найди подходящий компонент и **скопируй каноническую HTML-разметку** из доки
3. Добавь БЭМ-классы на обёртки/ключевые узлы; классы UIkit (`uk-*`, `uk-toggle` и data-атрибуты UIkit) оставь как в документации

Справка по типовым компонентам: [reference.md](reference.md#uikit).

### Контент и ассеты

- Тексты и иерархия заголовков — как в макете
- Изображения: `src` на `static/...` или плейсхолдер по имени из Figma; без `width`/`height`
- Ссылки-кнопки: `<a class="...">` или разметка UIkit для кнопок — по макету

## Правила SCSS

В SCSS файл напиши пустой шаблон со всеми классами, которые были в HTML. Сами стили писать не нужно. Только разметку

- Включай **все** БЭМ-классы из HTML
- Классы Swiper/UIkit (`swiper`, `swiper-slide`, `uk-*`) в SCSS **не дублируй**, если они не заданы как отдельные классы в HTML
- Не добавляй `@import`, переменные, свойства, вложенность со стилями
- Порядок селекторов — сверху вниз, как в HTML (блок → элементы → модификаторы)

Пример:

```scss
.hero {
  &__slider {}
  &__slide {
    &_first {}
  }
}
```

Подключение файла в `src/scss/style.scss` пользователь делает сам, если файл новый.

## Чеклист перед завершением

- [ ] Разметка соответствует макету (структура, тексты, количество повторяющихся элементов)
- [ ] У каждого элемента блока есть БЭМ-класс
- [ ] Нет `alt`, `aria-*`, `width`, `height`
- [ ] Слайдер на Swiper, прочий UI — по UIkit docs
- [ ] SCSS содержит пустые правила для всех БЭМ-классов из HTML
- [ ] В SCSS нет CSS-свойств
